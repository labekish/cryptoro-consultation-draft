# Локальная проверка черновика consultation

## 1) Запуск локального просмотра (без PHP)

```bash
cd /Users/leonidbekish/expert_session/local_draft/consultation
python3 -m http.server 8080
```

Откройте в браузере: `http://127.0.0.1:8080/preview.html`

## 2) Альтернатива для Bitrix/PHP-режима

Если установлен PHP:

```bash
php -S 127.0.0.1:8080
```

Откройте: `http://127.0.0.1:8080/index.php`

## 3) Визуальная проверка

- Блоки отображаются последовательно: hero → преимущества → этапы → аудитория → подготовка → форма → FAQ → стоимость → CTA.
- На мобильной ширине (DevTools, 390px) карточки и форма переходят в одну колонку.
- Hero-фото закруглено, без белой рамки.
- Секция «Что вы получите» — 2×2 карточки с фото, на мобиле стекают в столбец.
- Заголовок «Кто проводит консультации» переносится без обрезки.

## 4) Проверка интерактива

- Кнопка «Получить персональный план» скроллит к форме.
- Кнопка «Купить за 5 250 ₽» добавляет услугу в localStorage (`cryptoro_cart_draft`) и показывает тост.
- В FAQ открывается один пункт за раз, повторный клик закрывает его.
- В mock-форме проверяется обязательность полей «Имя», «Телефон», «Согласие».

## 5) Проверка конфиг-плейсхолдеров (для index.php)

Откройте `index.php` и проверьте значения переменных в начале файла:

| Переменная | Значение по умолчанию | Что задаёт разраб |
|---|---|---|
| `$CONSULT_FORM_ID` | `0` | ID веб-формы Битрикс |
| `$PRIVACY_URL` | `/privacy-policy/` | URL политики конфиденциальности |
| `$PAGE_SLUG` | `/consultation/` | Финальный путь страницы |

## 6) Что нужно разработчику при переносе в Bitrix

1. Задать реальный `$CONSULT_FORM_ID` — форма переключится с mock на `bitrix:form.result.new`.
2. Убедиться, что `$PRIVACY_URL` совпадает с реальным URL политики на сайте.
3. Подключить страницу в структуру сайта по нужному URL.
4. Проверить письма / обработчики формы Битрикс.
5. Реализовать реальную корзину (сейчас заглушка на localStorage, ключ `cryptoro_cart_draft`).
6. При необходимости зарегистрировать CSS/JS через `SetAdditionalCSS` / `AddHeadScript` вместо прямых `<link>`/`<script>` тегов.
