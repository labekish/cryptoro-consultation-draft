# Локальная проверка черновика consultation

## 1) Запуск локального просмотра (без PHP)

```bash
cd /Users/leonidbekish/expert_session/local_draft/consultation
python3 -m http.server 8080
```

Откройте в браузере: `http://127.0.0.1:8080/preview.html`

## 2) Альтернатива для Bitrix/PHP-режима

Если установлен PHP:

```bash
php -S 127.0.0.1:8080
```

Откройте: `http://127.0.0.1:8080/index.php`

## 3) Визуальная проверка

- Блоки отображаются последовательно: hero -> преимущества -> этапы -> аудитория -> подготовка -> форма -> FAQ -> стоимость -> CTA.
- На мобильной ширине (в DevTools, например 390px) карточки и форма переходят в одну колонку.
- Текст не выходит за границы карточек.

## 4) Проверка интерактива

- Кнопки `Получить экспертный разбор` и `Забронировать время` скроллят к нужным секциям.
- В FAQ открывается один пункт за раз, повторный клик закрывает его.
- В mock-форме проверяется обязательность полей `Имя`, `Телефон`, `Согласие`.

## 5) Проверка конфиг-плейсхолдеров (для index.php)

Откройте `index.php` и проверьте значения переменных в начале файла:

- `$CONSULT_FORM_ID` (ID веб-формы Битрикс, пока `0`)
- `$PRIVACY_URL` (ссылка на политику)
- `$PAGE_SLUG` (финальный путь страницы)

## 6) Что сделать при переносе в проект Bitrix

- Подставить реальный `$CONSULT_FORM_ID`.
- Подключить страницу в нужный URL в структуре сайта.
- Проверить, что `bitrix:form.result.new` выводится штатно и письма/обработчики формы работают.
